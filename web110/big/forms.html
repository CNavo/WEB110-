HTML Forms:

The purpose of HTML Forms:

HTML forms are required when you want to collect data from a user of your website.

What are HTML Forms exactly?

Forms are made up of interactive implements on a webpage the provider uses to gather information from the user. These “widgets” can consist of check boxes, lines to enter text, buttons and other interactives.

What elements go into creating a form?

Most forms are created with these basic elements:

  • <form> - All HTML forms start with this element. It a container element similar to <p> tags but supports specific attributes. The two you would use the most are 1) Action, which defines the location where the collected data should be sent and 2) Method, what you are sending the data with.
  • <label> - We use a “label” to apply each field we are requesting from the user. For instance, if we want to gather the users name the syntax may look something like this: <label for=”name”< Name:</label> or if we are requesting a text area for messages: <label for=”msg>”Message:</label> Remember user short but easy to understand labels for defining your input areas.
  • <input> - The area setup where the user will put in the information to continue with our label example: <input type=”text” id=”name” name=”user_name”>
  • <textarea> - A area to type out questions or comments by the user. Again, if we use our second label example: <textarea id=”msg” name=”user_message”></textarea>
  • <button> - A clickable widget that performs more or less “Submit” or “Reset” attributes. Example syntax for putting in a button:<button type=”submit”>Send your message</button>

Remember elements like label, input, button and text area all must be within the parent element of <form>

Summary and Other Resources:

Like the rest of your HTML web pages you want to utilize CSS to organize your forms and make them user friendly. Spacing, Padding, Margins and Borders help make your forms look sharp and easy for the user to understand what information is required from them. Be sure as a developer to use easy to understand labels for each requested field and always include a comment/message field to allow for feedback from the user on how you can improve accessibility.
