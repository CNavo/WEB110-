HTML ELEMENTS

Element Description Block, Inline, N/A Container or Empty
<!-- --> HTML comments that are invisible on a browser unless you view its source. N/A Container
<!DOCTYPE> The very first declaration you declare. Is is instruction to the web broswer on how the file is written. N/A Empty
<a> This tag provides a hyperlink from one page to another Inline Container
<article> When used, this tag provides specialized content that can be independent from the rest of the page. N/A Container
<aside> You use the aside tag when you want to add related content from its main content N/A Container
<body> The area of your page that includes the majority of your content. such as images, tables, text etc. N/A Container
<br/> The br/ or line break tag will provide an a single line break between text. Often used in writing addresses. Inline Empty
<dd> This tag is used to decribe a term or name in a description list. Block Container
<div> The div is sometimes used to a selected area of the page and set it with specific styling and layout. Block Container
<dl> Used often with dd, the dl element defines a description list. Block Container
<dt> Used often with dd and dl, the dt element defines the names and terms in a description list. Block Container
<em> Marks text with emphasis. Helpful for broswer readers on where to stress that emphasis when reading the file. Inline Container
<fieldset> This tag groups related elements in a form that draws a box around those in relation. Block Container
<figcaption> This element will provide a caption or definition for images on the page N/A Container
<figure> This tag will specify self-contained content while remaining independent of the main source, so removing it will not affect the page. N/A Container
<footer> Seen often at the end of content, the footer is used to reference information such as related information, copyrights, and navigating back to the top of the page. N/A Container
<form> This tag is used to create a form for user input. They will often provide interactive areas such as checkboxes and textfields. Block Container
<h1> to <h6> HTML provides 6 different level of headers h1 for instance is often only used once per document as it often acts as the main header. As the number of the header increase the emphasis of the header decreases. Block Container
<head> This element includes all the head elements such as your title, you CSS links, other html links, and your meta tags. N/A Container
<header> Often contains your logo or introductory can often include your h1-h6 elements. N/A Container
<html> This element provides a refernce to the majority of the page referencing the anything in the tag is an HTML document. N/A Container
<iframe> This element allows an external webpage to be embedded in an HTML document. Inline and Block Container
<img/> This element defines an image on an HTML page. Images are linked to HTML pages the img tag creates a space for it. Inline Empty
<input> The input tag specifies a field where the user can enter in data. Inline Container
<label> The label defines a text label for the input tag. The label is ordinary text, when clicked on by the user can select the form element. Inline Container
<li> This element represents a list item in ordered and unordered list. Often used with the menu tag. Block Container
<link/> Short for hyperlink, link will allow a user to jump to a new location when you click or tap it. Links are often used on every webpage. N/A Empty
<meta/> Found primarily in the header, meta elements are used to specify the page description, it is used by browsers, search engines, and other web servers. The data in a meta is not displayed on the page. N/A Empty
<nav> A brand new element to HTML5, it is used to define a block of navigation links within the current page or other documents. N/A Container
<ol> ol or ordered list, it is often used with a li and can be numerical or alphabetical. Block Container
<p> The p or paragraph tag is used whenever you want to relay information or text on a webpage. It can be modified using CSS properties. Block Container
<script> This element either contains scripting statments pr directs to an external file using the src attribute. Inline and Block Container
<section> Different from the div tag which groups blocked elements, the sections element defines those sections such as chapters, headers, and footers. N/A Container
<strong> Emphasises text with importance, and often uses bold text seen by the user. If you want browser readers to read something with emphasis the em element would be used instead. Inline Container
<style> This tag is used to define the style of the HTML document. Often this will include a link to a seperate CSS that will contain those styles wanted for the page. N/A Container
<table> A structure element that builds a table of rows and columns on a Web page. tr tags build each row while td tags define the data. Block Container
<td> The td element stands for table data and is a standard cell in a HTML table. These standard cells are within tr and often contain text data. Block Container
<th> The th element stands for table header. They contain the header of the contents found in each column of a table. Block Container
<title> A requirement of all HTML documents. It will define the title in the toolbar of a webpage, making it easier for a user to add to favorites. N/A Container
<tr> The tr element stands for table row. The cells inside are defined using the th (header cell) or a td (standard data cell) Block Container
<ul> ul or unordered list, this tag will bullet lists as opposed to alphbetize or numbered list. Block Container
